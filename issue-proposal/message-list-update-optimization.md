# Оптимизация обновления UI при изменении статуса сообщений

## Описание

В текущей архитектуре изменение статуса сообщения (например отправлено, доставлено, прочитано) приводит к обновлению записи в базе данных.

Это обновление распространяется через реактивный pipeline приложения.

Пример pipeline:

database update  
↓  
signal emission  
↓  
пересчёт состояния чата  
↓  
обновление списка сообщений  
↓  
обновление UI

При этом изменение касается только небольшой части интерфейса — индикатора статуса сообщения.

---

## Сценарий

1. Пользователь отправляет сообщение
2. Статус сообщения последовательно меняется:

sending → sent → delivered → read

Каждое изменение статуса вызывает обновление записи в базе данных.

Хотя визуально меняется только индикатор статуса, UI может выполнять более широкий pipeline обновления.

---

## Влияние

### Производительность

В активных чатах частые обновления статусов сообщений могут приводить к дополнительной нагрузке на UI pipeline.

### Пользовательский опыт

В больших чатах это может вызывать микрофризы при быстром поступлении обновлений.

---

## Предложение решения

Добавить более гранулярный механизм обновления UI.

При изменении статуса сообщения обновлять только компонент индикатора статуса, не пересчитывая всю message cell.

Пример структуры message cell:

```
MessageCell
├── текст сообщения
├── время
└── deliveryStatusView
```

В случае изменения статуса должен обновляться только `deliveryStatusView`.

## Возможная реализация
Ввести более гранулярное обновление UI. Использовать тип обновления:
```swift
enum MessageUpdateType {
    case contentChanged
    case statusChanged
    case reactionChanged
}
```
При изменении статуса обновлять только `deliveryStatusView` вместо всей ячейки.

---

## Стратегия тестирования

Функциональные тесты:

- отправка сообщений
- получение статуса доставки
- получение статуса прочтения

Performance тесты:

- чат с большим количеством сообщений
- частые обновления статусов

Метрики:

- время обновления UI
- количество обновлённых message cell
- frame drop rate

---
